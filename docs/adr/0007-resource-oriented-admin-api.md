# Use One Resource-Oriented Admin API With Distinct Authentication Adapters

Status: accepted

Caplets will expose Current Host administration through one versioned, resource-oriented Admin API. The canonical Operator-bearer surface is `/v2/admin`; the dashboard mounts the same relative resources at `/dashboard/api/v2` behind its existing cookie, session, and CSRF ceremony. Both Adapters invoke the same `CurrentHostOperations` semantic Module. Runtime tools, resources, prompts, and execution remain on Attach, node-filesystem `init` and `add` remain local-only, and Raw Vault Reveal remains a dashboard-private human ceremony.

A public, cacheable `/openapi.json` at the service root will describe every canonical public v1 and v2 HTTP API except MCP, dashboard cookie/session/CSRF routes, Raw Vault Reveal, and other browser-private routes. Route-local Zod schemas generate OpenAPI 3.1, and an exactly pinned HeyAPI toolchain generates the root Fetch client in the independent public `@caplets/sdk` package after proving Caplets' ordered multipart upload and streaming multipart download contracts. Successful v2 responses use HTTP resource representations; failures use Problem Details with a stable Caplets code. Mutable resources use ETags and conditional requests, growable collections use cursor pages, and retryable POSTs use durable principal-scoped idempotency records.

Before the first public client release, generated-client ownership cuts over cleanly to `@caplets/sdk`; core retains no public client export or compatibility alias. The SDK root combines the browser/Node generated client with curated Caplet Bundle helpers. `@caplets/sdk/project-binding` owns the browser-safe coordinator for the fixed `caplets.project-binding.v1` protocol, whose `ProjectBindingSocketClientMessage` and `ProjectBindingSocketServerMessage` components are named public schemas in the canonical OpenAPI document. `@caplets/sdk/project-binding/node` owns the marker-aware filesystem fingerprint helper.

SDK callers create isolated clients, provide an explicit service root, and optionally supply caller-owned static or async authentication. The Project Binding coordinator separately requires an exact `ws:` or `wss:` connect URL and a caller-supplied project fingerprint; it does not infer endpoints or place bearer credentials in URLs. These package-boundary decisions amend the delivery shape without changing this ADR's accepted resource model or distinct Operator-bearer and dashboard authentication ceremonies.

Full Caplet Bundles cross the API as manifest-first streaming `multipart/form-data` uploads and streaming `multipart/mixed` downloads. Uploads are admitted and bounded per Host Node, staged in process-owned ephemeral storage, verified before semantic mutation, and imported through reopenable bundle sources. Pending backend OAuth flows and idempotency records live in Authoritative Host State so different Host Nodes preserve one coherent administrative view.

`POST /v1/admin` remains available without a scheduled removal date as a deprecated, frozen compatibility Adapter. It gains no commands and delegates to the same semantic operations or Attach execution paths; remote `init` and `add` are rejected when v2 launches. This preserves deployed clients without retaining a second policy implementation. The rejected alternatives were a mixed-auth route tree, direct store-backed handlers, permanent dual evolution of v1 and v2, JSON/base64 bundle transport, node-local OAuth flows, and generated clients coupled to Hono RPC types.
