# Caplets Docs Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use `subagent-driven-development` or `executing-plans` to implement this plan task-by-task. Use `$impeccable` during UI/theme work and before final completion. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the public Caplets documentation site at `docs.caplets.dev`, deployed beside `apps/landing` through Alchemy.

**Architecture:** Add `apps/docs` as the canonical public docs app. Keep root `docs/` internal for architecture, plans, specs, PRDs, ADRs, and maintainer material. Deploy landing and docs from the root `alchemy.run.ts`, with shared domain generation in `infra/alchemy-domains.ts`.

**Tech Stack:** Astro, Starlight, MDX, Tailwind v4, optional Starwind for components beyond Starlight, Alchemy Cloudflare Astro deploys, pnpm, Turbo, Changesets.

---

## Decisions

- Public docs live in `apps/docs`.
- Internal docs remain in root `docs/`.
- Do not sync or route root `docs/` content into the public site.
- Teach Code Mode as the default Caplets surface.
- Show both `npx caplets ...` and global install examples.
- Use `osv` as the first no-auth runnable example.
- Use Starlight components first, local Astro components second, Starwind only when Starlight is insufficient.
- Use Tailwind for custom styling unless Starlight compatibility is better served by Starlight CSS variables.
- Keep docs latest-only for pre-1.0, but avoid hardcoded page links that would block future versioned routes.
- Keep canonical schema URLs on `https://caplets.dev/config.schema.json` and `https://caplets.dev/caplet.schema.json`.
- Exclude `@caplets/docs` from Changesets.
- Include `@caplets/docs` in build/typecheck/verify.

    return {

      candidates: matches.items.map((item) => item.name),
