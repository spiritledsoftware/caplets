# Landing Starwind Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use subagent-driven-development (recommended) or executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Convert `apps/landing` from a mostly custom Astro/CSS page to an Astro-native Starwind UI + Tailwind CSS implementation while preserving the Caplets palette, product story, and major visual identity.

**Architecture:** Starwind owns generic UI primitives and theme tokens; Tailwind utilities own layout and most styling; custom CSS remains only for Caplets-specific page background, complex Three.js/canvas masking, and any browser behavior Tailwind cannot express cleanly. The current monolithic `src/pages/index.astro` is split into data, layout, section components, and small scripts by concern.

**Tech Stack:** Astro 6, Tailwind CSS 4, Starwind UI, Three.js, Tabler icons through Starwind defaults, pnpm 11.5.0, Node 24+.

---

## Locked Decisions

- Use Starwind, not React shadcn.
- Preserve the current Caplets palette and style identity, but accept Starwind-driven shape and spacing changes if the site still clearly reads as Caplets.
- Use Starwind primitives only. Do not use Starwind Pro/page blocks.
- Split `src/pages/index.astro` conservatively into focused section components.
- Full dark-mode cutover to Starwind's `.dark` convention. Remove `data-theme`.
- Be very aggressive about removing vanilla CSS. Keep it only where Tailwind or Starwind cannot carry the behavior cleanly.
- Use small Astro wrappers only when they remove real repetition or isolate behavior.
- Remove HugeIcons completely and use Starwind's default icon path.
- Use Starwind `Theme Toggle`.
- Replace the native mobile nav dialog with Starwind `Sheet`.
- Use Starwind `Tabs` for Caplet examples. Keep hero city mode controls custom because they are coupled to Three.js behavior.
- Simplify copy behavior. Keep high-value copy actions only.
- Split browser scripts by concern.
- Reimplement reveal motion with Tailwind/Starwind-compatible patterns.
- Browser-verify the migration on desktop and mobile.
- Delete `apps/landing/test/campaign-copy.check.mjs` and remove the `campaign:check` script.
- Keep `apps/landing/test/schema-assets.test.ts`.
- Put Starwind/system CSS in `src/styles/starwind.css`; keep `src/styles/global.css` for Caplets-specific leftovers.
- Use `starwind init`, inspect generated files, then clean them up into the Caplets theme.
- Add likely Starwind components upfront, then prune unused generated components before final verification.

## Product And Design Constraints

Use the project context from `PRODUCT.md` and `DESIGN.md`:

- Caplets is precise, calm, capable, and developer-tool focused.
- The landing page must keep the `Code Mode`-first story and exact visible naming.
- The palette remains ember, parchment, charred ink, linen, paper, ash, and muted olive.
- Ember stays rare. Use it for selected states, primary action hover, and key signals, not decoration.
- Avoid generic SaaS cream, hero metric cliches, identical card grids, decorative gradients, neon devtool dark mode, and generic component-demo output.
- Keep full keyboard navigation, visible focus states, readable contrast, non-color-only state cues, and reduced-motion behavior.

```ts
const copyButtons = Array.from(document.querySelectorAll<HTMLButtonElement>("[data-copy-value]"));
const copyStatus = document.querySelector<HTMLElement>("[data-copy-status]");

function setCopyFeedback(button: HTMLButtonElement, state: "copied" | "failed") {
  const label = button.dataset.copyLabel ?? "value";
  button.dataset.copyState = state;
  button.setAttribute("aria-live", "polite");
  if (copyStatus) {
    copyStatus.textContent = state === "copied" ? `Copied ${label}.` : `Could not copy ${label}.`;
  }
  window.setTimeout(() => {
    delete button.dataset.copyState;
    button.removeAttribute("aria-live");
  }, 1600);
}

async function copyValue(button: HTMLButtonElement) {
  const value = button.dataset.copyValue;
  if (!value) return;

  try {
    await navigator.clipboard.writeText(value);
    setCopyFeedback(button, "copied");
  } catch {
    setCopyFeedback(button, "failed");
  }
}

for (const button of copyButtons) {
  button.addEventListener("click", () => void copyValue(button));
}
```

```ts
const targets = Array.from(document.querySelectorAll<HTMLElement>("[data-reveal]"));
const reduceMotion = window.matchMedia("(prefers-reduced-motion: reduce)").matches;

if (targets.length > 0 && !reduceMotion && "IntersectionObserver" in window) {
  const observer = new IntersectionObserver(
    (entries) => {
      for (const entry of entries) {
        if (!entry.isIntersecting) continue;
        const target = entry.target as HTMLElement;
        target.dataset.reveal = "visible";
        observer.unobserve(target);
      }
    },
    { rootMargin: "0px 0px -10%", threshold: 0.12 },
  );

  for (const target of targets) {
    target.dataset.reveal = "pending";
    observer.observe(target);
  }

  window.setTimeout(() => {
    for (const target of targets) {
      if (target.dataset.reveal === "pending") target.dataset.reveal = "visible";
    }
  }, 240);
} else {
  for (const target of targets) {
    target.dataset.reveal = "visible";
  }
}
```


Content must be visible by default before JS sets `data-reveal="pending"`.
